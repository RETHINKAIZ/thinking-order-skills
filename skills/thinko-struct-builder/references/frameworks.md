## 一、通用结构化表达框架

### 1. 金字塔原理

**结构**：中心思想 → 支撑思想 → 论据（纵向延伸 + 横向并列）

**使用场景**：所有需要逻辑清晰、结论先行的沟通场景

**示例**：
```
中心思想：应该采用 A 方案
├─ 支撑思想 1：A 方案成本更低
│  ├─ 论据：实施成本降低 30%
│  └─ 论据：维护成本降低 50%
├─ 支撑思想 2：A 方案效率更高
│  ├─ 论据：处理速度提升 2 倍
│  └─ 论据：错误率降低至 1%
```

### 2. MECE 原则

**结构**：相互独立（Mutually Exclusive）、完全穷尽（Collectively Exhaustive）

**使用场景**：信息分类、问题分析、避免遗漏或重叠

### 3. 5W2H

**结构**：Why→What→Who→When→Where→How→How much

**使用场景**：计划制定、任务准备

### 4. 5WHY 分析法

**结构**：连续追问"为什么"直至根本原因

**使用场景**：探究问题根源

## 二、汇报工作场景框架

### 1. PREP 框架

**结构**：
1. **Point（结论）** - 先说结论
2. **Reason（依据）** - 说明理由
3. **Example（事例）** - 举例佐证
4. **Point（重申结论）** - 再次强调

**模板**：
```
【结论】我的观点/建议是：___________

【依据】理由是：
- 第一，___________
- 第二，___________

【事例】例如：___________

【重申】因此，我建议：___________
```

### 2. STAR 框架

**结构**：
1. **Situation（背景）** - 当时的情况
2. **Task（任务）** - 我的任务/目标
3. **Action（行动）** - 我做了什么
4. **Result（结果）** - 取得了什么成果

**模板**：
```
【背景】当时的情况是：___________

【任务】我需要完成的任务是：___________

【行动】我采取的行动包括：
- 首先，___________
- 然后，___________
- 最后，___________

【结果】最终成果：
- 定量成果：___________
- 定性收获：___________
```

### 3. 层形结构

**结构**：背景 → 形势变化 → 核心建议

**适用**：受众对背景不熟悉或可能抵触建议时

### 4. 列形结构

**结构**：核心建议 → 并列支撑要点

**适用**：受众熟悉主题时（如向上级汇报）

## 三、客户沟通场景框架

### 1. FABE 框架

**结构**：
1. **Feature（特点）** - 产品特性
2. **Advantage（优点）** - 相比竞品的优势
3. **Benefit（利益）** - 给客户带来的价值
4. **Evidence（证据）** - 证明以上各点的证据

**模板**：
```
【特点】这款产品的特点是：___________

【优点】这意味着/相比其他产品的优势是：___________

【利益】对您来说，这能带来：
- ___________
- ___________

【证据】我们的证明包括：
- 数据：___________
- 案例：___________
- 认证：___________
```

### 2. AIDA 框架

**结构**：Attention→Interest→Desire→Action

**模板**：
```
【注意】您有没有遇到过这样的情况：___________

【兴趣】我们有一个解决方案，它可以：___________

【欲望】想象一下，如果您能够：___________

【行动】现在，您可以：___________
```

### 3. SPIN 框架

**结构**：
1. **Situation（现状问题）** - 了解客户现状
2. **Problem（难点问题）** - 挖掘客户痛点
3. **Implication（暗示问题）** - 放大痛点影响
4. **Need-payoff（解决型问题）** - 引导客户说出解决方案

**模板**：
```
【现状】您目前是如何处理___________的？

【难点】在这个过程中，您遇到的最大挑战是什么？

【影响】这个问题如果不解决，会对___________产生什么影响？

【解决】如果有办法解决这个问题，对您意味着什么？
```

## 四、讲故事场景框架

### 1. 完整 7 步结构

**结构**：目标 → 阻碍 → 努力 → 结果 → 意外 → 转折 → 结局

**模板**：
```
【目标】主人公想要：___________

【阻碍】但是遇到了：___________

【努力】于是他：___________

【结果】得到了：___________

【意外】然而意外发生了：___________

【转折】关键时刻：___________

【结局】最终：___________
```

### 2. "努力人"框架

**结构**：目标 → 阻碍 → 努力 → 结果

**模板**：
```
【目标】我想要：___________

【阻碍】但是我面临：___________

【努力】于是我坚持：___________

【结果】最终我做到了：___________
```

### 3. "意外人"框架

**结构**：目标 → 意外 → 转机 → 结果

**模板**：
```
【目标】我原本计划：___________

【意外】没想到发生了：___________

【转机】但我抓住机会：___________

【结果】意外收获了：___________
```

## 六、写作场景框架

### SCQA 框架

**结构**：Situation→Complication→Question→Answer

**模板**：
```
【情境 S】我们都知道/目前的情况是：___________

【冲突 C】然而/但是：___________

【疑问 Q】那么/我们应该如何解决？___________

【回答 A】我们的方案是：___________
```

### 序言类型选择

| 类型 | 结构 | 适用 |
|------|------|------|
| 提出方案型 | S→C→Q→A | 方案申请、立项报告 |
| 实施方案型 | S→C→A | 执行计划、操作指南 |
| 分析问题型 | S→C→Q | 调研报告、问题分析 |
| 定义问题型 | S→C | 问题描述、情况说明 |
